# How to write `to-do.md` entries

**What this doc is for:** the format for `to-do.md`, the platform backlog. The
backlog is a short list of things to investigate or build *later* — deliberately
low-detail, so it stays a quick scan and doesn't rot into a second source of
truth. Live state lives in code and each epic's `ISSUES.md`; this is only the
"don't forget to look at this someday" list.

## The format

One entry per item. An entry is just a **title** and **one or two sentences** —
enough to recognise the item and why it mattered, not how to do it. If you find
yourself writing specifics (file names, commands, a design), the item has
outgrown the backlog and belongs in an epic's `ISSUES.md` instead.

Newest or most relevant at the top. Delete an entry once it's done or promoted
to an epic — the backlog should only ever hold things not yet started.

## Example

```
## Short imperative title

One sentence on what it is. One sentence on why it matters or what to look at.
```
